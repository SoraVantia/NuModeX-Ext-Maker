# NuModeX Ext Maker スタートガイド

このガイドでは、セットアップ、初回ビルド、よくある問題についてREADMEよりも詳しく説明します。

## 前提条件

- Chromiumベースのブラウザ（Chrome、Edge、Brave、Whale）またはFirefox
- 少なくとも1つのクラウドAIプロバイダーのAPIキー - オンデバイスモデルを使用する場合は不要
- ブラウザ拡張機能の基本的な知識（あると便利ですが必須ではありません）

## クラウドモデルのセットアップ

### 1. 拡張機能のインストール

**ブラウザ拡張機能ストアから（近日公開予定）:**
NuModeX Ext Makerはまだどのブラウザ拡張機能ストアでも公開されていません。ストアでの公開は [numodex.com/numodexextmaker](https://numodex.com/numodexextmaker) およびGitHub Releasesで発表されます。

**ソースから（デベロッパーモード）:**
1. リポジトリをクローンまたはダウンロード。
2. `chrome://extensions`（またはお使いのブラウザの同等のページ）を開く。
3. デベロッパーモードを有効にする。
4. 「パッケージ化されていない拡張機能を読み込む」をクリックし、適切なブラウザフォルダ（`browsers/chrome`、`browsers/edge`、または`browsers/firefox`）を選択。

### 2. APIキーの取得

少なくとも1つのサポートされたクラウドAIプロバイダーからAPIキーが必要です。プロバイダーの開発者コンソールまたはAPIプラットフォームにアクセスしてキーを生成してください。各プロバイダーのキーは拡張機能内で個別に保存されます - プロバイダー間の切り替えは自由です。

### 3. 設定とビルド

1. 利用規約に同意（初回起動時に自動的に表示されます）。
2. 拡張機能ポップアップの設定アイコンをクリック。
3. APIキーを貼り付けて「キーを保存」をクリック。
4. ドロップダウンからAIモデルを選択。
5. チャットで構築したいものを説明。
6. 「拡張機能をビルド」または「ウェブサイトをビルド」をクリックして生成を待つ。
7. 内蔵の編集ツールを使って、必要に応じて生成されたファイルをレビュー・編集。
8. 「すべてをZIPでダウンロード」をクリック。
9. 拡張機能の場合：ZIPを展開し、`chrome://extensions`にアクセスし、デベロッパーモードを有効にして「パッケージ化されていない拡張機能を読み込む」をクリック。ウェブサイトの場合：展開して`index.html`をブラウザで開く。

> **その他のブラウザ：** 生成された拡張機能はManifest V3で、Edge、Brave、Whale、その他のChromiumベースのブラウザと互換性があります。サイドロードの手順はブラウザによって異なります。

## オンデバイスモデルのセットアップ

オンデバイスモデルはAPIキーやクラウド接続なしで、完全にお使いのハードウェア上で動作します。**これらのモデルは特定のブラウザでのみ利用可能です：** Google ChromeではGemini Nano、Microsoft EdgeではPhi-4 Mini。その他のChromiumベースのブラウザ（Brave、Whaleなど）やFirefoxは、現在ブラウザAPIを通じたオンデバイスAIをサポートしていません。

**クラウドモデルとの主な違い：**
- オンデバイスモデルは**チャットとファイル編集**にのみ使用でき、フルビルドには使用できません。
- 初回使用時にモデルのダウンロードが必要です（数分かかる場合があります）。
- ハードウェア要件が厳しいため、トラブルシューティング前に確認してください。

### Chrome - Gemini Nano

1. Chromeバージョン127以上を使用（DevまたはCanaryを推奨）。
2. `chrome://flags/#optimization-guide-on-device-model` にアクセスし、**Enabled BypassPerfRequirement** に設定。
3. `chrome://flags/#prompt-api-for-gemini-nano` にアクセスし、**Enabled** に設定。
4. Chromeを再起動。
5. `chrome://on-device-internals` にアクセスしてモデルのステータスを確認。モデルがダウンロードされていない場合は、`chrome://components/` にアクセスし、**Optimization Guide On Device Model** を見つけて **Check for update** をクリック。
6. モデルのダウンロードを待つ。数分かかる場合があります。ダウンロード中はChromeを開いたままにしてください。

### Edge - Phi-4 Mini

1. Edge DevまたはCanary（バージョン138以上）を使用。Edge 139以降ではPhi-4 Miniがデフォルトで含まれています。
2. `edge://flags/` にアクセスし、**Prompt API for Phi mini** を検索して **Enabled** に設定。
3. オプションで **Enable on device AI model debug logs** を有効にしてトラブルシューティングに役立てる。
4. Edgeを再起動。
5. `edge://on-device-internals` にアクセスし、**Device performance class** が **High** 以上であることを確認。
6. モデルは初回使用時に自動的にダウンロードされます。数分かかる場合があります。ダウンロード中はEdgeを開いたままにしてください。

### ハードウェア要件

**Edge:** Windows 10/11またはmacOS 13.3以降、20 GB以上の空き容量、5.5 GB以上のVRAM、従量制でないインターネット接続。

**Chrome:** 22 GBの空き容量、4 GB超のVRAM（GPU）または16 GB以上のRAMと4コア以上のCPU（CPUモード）、従量制でない接続。

> **注意：** オンデバイスモデルはチャットとファイル編集にのみ使用できます。拡張機能やウェブサイトの完全なビルドにはクラウドモデルを選択してください。

## ビルドモードについて

NuModeX Ext Makerには2つのビルドモードがあります：

**拡張機能をビルド** - manifest.json、ポップアップファイル、コンテンツスクリプト、その他必要なファイルを含む完全なManifest V3ブラウザ拡張機能を生成します。

**ウェブサイトをビルド** - HTML、CSS、JavaScriptファイルを含む完全な静的ウェブサイトを生成します。

両モードとも同じチャットインターフェースを使用します。AIは生成時に会話履歴全体を読み取るため、ビルド前に複数のメッセージで要件を詳細化できます。

## 生成されたファイルの確認

ビルド完了後、拡張機能の右パネルに生成されたファイルが表示されます。

**ファイルツリー** - 生成されたすべてのファイルが右パネル上部のクリック可能なリストに表示されます。ファイルをクリックすると内容を表示できます。ファイルはサブディレクトリパス（例：`assets/style.css`）を含む名前で整理されています。

**コードビューア** - ファイルを選択すると、ファイルツリーの下のコードビューアにシンタックスハイライト付きで内容が表示されます。ビューアはファイルタイプ（JavaScript、JSON、HTML、CSS、Markdown）を自動検出し、適切なハイライトを適用します。コピーボタンを使って、任意のファイルの内容をクリップボードにコピーできます。

**手動編集** - コードビューア上部の編集トグルをクリックして、手動編集モードに切り替えます。コードビューアがテキストエディタに変わり、直接手で変更を加えることができます。再度トグルをクリックすると手動編集モードを終了します。

**ライブプレビュー** - 拡張機能やウェブサイトのビジュアルプレビューを見るには、その他 (▾) > プレビューをクリックします。サンドボックス化されたプレビューがモーダルで開き、popup.html（拡張機能の場合）またはindex.html（ウェブサイトの場合）をレンダリングします。プレビューはCSSとJavaScriptファイルを自動的にインライン化して正しくレンダリングします。これはビジュアルのみのプレビューです - ブラウザ拡張機能API（chrome.tabs、chrome.storageなど）や外部リソースはプレビューでは動作しません。プロジェクトにpopup.htmlやindex.htmlがない場合、プレビューは情報メッセージを表示します。

**変更の表示** - AIによる編集後、変更の表示をクリックすると、変更前後の差分を確認できます。統合ビューとサイドバイサイドビューを切り替えることができます。複数のファイルが変更された場合、上部のタブで切り替えられます。

## ビルド後の編集

初回ビルド後、いくつかの編集オプションがあります：

**ファイルを編集** - 単一のファイルを選択して変更を記述します。ターゲットを絞った修正に最適です。

**ファイルを追加** - 新しいファイルを作成し、含めるべき内容を記述します。

**拡張機能を改善** - プロジェクト全体にわたる変更を記述します。AIは複数のファイルを一度に変更できます。

**手動編集** - コードビューアに直接クリックして手動でコードを編集します。

**元に戻す** - 最後のAI編集を元に戻します。元に戻せるのは1回分のみです。

## 適切なモデルの選択

特定のモデルを推奨するのではなく（頻繁に変更されるため）、重要な要素に基づいた選び方を説明します：

**コンテキストウィンドウサイズ** - モデルが一度に処理できる会話履歴とコードの量を決定します。少数のファイルで構成されるシンプルな拡張機能には、小さなコンテキストウィンドウで十分です。複雑なマルチファイルプロジェクトには、すべてのファイルと会話を一度に参照できるよう、大きなコンテキストウィンドウを持つモデルを選択してください。

**出力トークン制限** - モデルが1回のレスポンスで生成できるコードの量を決定します。JSONパースエラーや出力の切り捨てが発生する場合は、より高い出力制限を持つモデルに切り替えてください。大きなプロジェクトにはより多くのトークンを出力できるモデルが必要です。

**推論能力** - 複雑なロジック、マルチステップの指示、ファイル間の一貫性の維持に優れたモデルがあります。シンプルなポップアップ拡張機能にはほとんどのモデルが適しています。コンテンツスクリプト、バックグラウンドワーカー、複雑なインタラクションを持つ拡張機能には、強力な推論能力を持つモデルを選択してください。

**コスト** - 大きく高性能なモデルはAPIコールごとのコストが高くなります。チャットやシンプルな編集には小さなモデルから始め、複雑なプロジェクトのビルドや改善が必要な場合に大きなモデルに切り替えてください。コストは選択するモデルとAIプロバイダーによって異なります。SoraVantia合同会社はいかなるAIプロバイダーとも提携しておらず、API料金の一部を管理または受け取ることはありません。

**速度** - 小さなモデルはレスポンスが速くなります。小さな変更を素早く反復する場合は、速いモデルが時間を節約します。品質が速度よりも重要なフルビルドには、大きなモデルを使う価値があります。

| ユースケース | 重視すべきポイント |
|----------|-----------------|
| シンプルな拡張機能（ポップアップのみ、小さなスクリプト） | 適度なコンテキストと出力制限を持つ任意のモデル |
| 複雑な拡張機能（コンテンツスクリプト、バックグラウンドワーカー、複数ファイル） | 大きなコンテキストウィンドウ、高い出力制限、強力な推論能力 |
| クイックチャットの質問やブレインストーミング | 任意のモデル - 出力サイズよりも速度が重要 |
| 単一ファイルの編集 | オンデバイスモデルを含む任意のモデル（無料、APIキー不要） |
| コスト重視の開発 | チャットには小型または中型モデル、ビルドにのみ大型モデル |

AIプロバイダーのモデルドキュメントで、現在のコンテキストウィンドウサイズ、出力制限、価格を確認してください。

## より良い結果のためのヒント

- シンプルな説明から始めて徐々に構築。まずコア機能を説明し、その後「編集」と「改善」を使って段階的に機能を追加。
- 複雑なプロジェクトにはコンテキストウィンドウが大きいモデルを使用。大きなモデルは小さなモデルよりも大きな出力をうまく処理します。
- 「拡張機能ファイルを抽出できませんでした」と表示された場合、プロンプトが1回の生成には複雑すぎます。初期プロンプトをシンプルにし、編集を通じて機能を追加してください。
- JSONパースエラーが表示された場合、モデルの応答が長すぎて途中で切れています。よりシンプルなプロンプトを試すか、出力制限が大きいモデルに切り替えてください。
- クラウド、カスタム、リモートモデルはすべてビルド、編集、チャットに使用できます。ニーズと予算に最適なモデルを選択してください。
- オンデバイスモデルはチャットと編集に対応していますが、完全な拡張機能やウェブサイトのビルドはできません。ビルドにはクラウドまたはカスタムモデルを使用してください。
- Enterでチャットメッセージを送信。Shift+Enterで改行。Ctrl/Cmd+Enterで拡張機能をビルド。Ctrl/Cmd+Shift+Enterでウェブサイトをビルド。
- ビルド後、単一ファイルの変更には「ファイルを編集」、複数ファイルにまたがる変更には「拡張機能を改善」を使用。
- その他 (▾) > ファイルをインポートから既存のファイルをインポートしてAIで編集。

## トラブルシューティング

**トラブルシューティングの前に、以下のリソースを確認してください：**
- クラウドAIプロバイダーのAPIドキュメントで、現在のモデル制限、価格、ステータスを確認。
- ブラウザの開発者ドキュメントで、最新のオンデバイスAI要件と既知の問題を確認。
- 上記の[オンデバイスモデルのセットアップ](#オンデバイスモデルのセットアップ)セクションでフラグとハードウェア要件を確認。

### 「APIキーが設定されていません」
設定を開き、選択したプロバイダーのAPIキーを貼り付けてください。各プロバイダーには独自のキーがあります - 選択したモデルのプロバイダーのキーを保存したか確認してください。

### 「拡張機能ファイルを抽出できませんでした」
AIの応答が複雑すぎるか不正な形式です。以下を試してください：
- プロンプトをシンプルにする（一度に少ない機能を記述）
- より大きな出力制限を持つモデルに切り替える
- まず基本バージョンをビルドし、その後「拡張機能を改善」で機能を追加

### 「JSONパースエラー」
モデルの応答が長すぎて、JSONが完成する前に切り捨てられました。以下を試してください：
- よりシンプルなプロンプト
- より高い出力トークン制限を持つモデル

### オンデバイスモデルが「ダウンロード中」のまま止まる
よくある問題です。以下を確認してください：
1. **ハードウェア要件を満たしていますか？** `edge://on-device-internals`（Edge）または`chrome://on-device-internals`（Chrome）にアクセスしてデバイスクラスを確認。
2. **フラグが有効ですか？** 上記の[オンデバイスモデルのセットアップ](#オンデバイスモデルのセットアップ)セクションを参照。
3. **十分なストレージがありますか？** Edgeは20 GB、Chromeは22 GBの空きが必要です。
4. **従量制接続ですか？** 従量制/データ制限のある接続ではモデルはダウンロードされません。
5. **ブラウザを開いたままにしていますか？** ブラウザを閉じるとダウンロードは停止します。
6. **ブラウザを再起動して** 5-10分待ってみてください。

### ロード後に拡張機能が動作しない
- ブラウザコンソール（`chrome://extensions` > 拡張機能の「エラー」をクリック）でエラーメッセージを確認。
- 正しいフォルダ（manifest.jsonを含むフォルダ）をロードしたか確認。
- 正確な動作を指定するより詳細なプロンプトで再生成を試す。

### カスタムモデルが応答しない
- エンドポイントURLが正しく、到達可能であることを確認。
- サーバーが `/v1/chat/completions` API形式をサポートしていることを確認。
- サーバーがAPIキーを必要とするか、また提供したか確認。
- プロンプトが表示されたら、拡張機能のホスト権限を許可。
