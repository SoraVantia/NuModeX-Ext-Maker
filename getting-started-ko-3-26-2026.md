# NuModeX Ext Maker 시작 가이드

이 가이드에서는 설정, 첫 번째 빌드 및 일반적인 문제에 대해 README보다 자세히 설명합니다.

## 전제 조건

- Chromium 기반 브라우저(Chrome, Edge, Brave, Whale) 또는 Firefox
- 최소 하나의 클라우드 AI 제공업체의 API 키 - 온디바이스 모델을 사용하는 경우 불필요
- 브라우저 확장 프로그램에 대한 기본 지식 (도움이 되지만 필수는 아님)

## 클라우드 모델 설정

### 1. 확장 프로그램 설치

**브라우저 확장 프로그램 스토어에서 (출시 예정):**
NuModeX Ext Maker는 아직 어떤 브라우저 확장 프로그램 스토어에서도 제공되지 않습니다. 스토어 출시는 [numodex.com/numodexextmaker](https://numodex.com/numodexextmaker) 및 GitHub Releases에서 발표될 예정입니다.

**소스에서 (개발자 모드):**
1. 저장소를 클론하거나 다운로드.
2. `chrome://extensions` (또는 사용 중인 브라우저의 해당 페이지)를 열기.
3. 개발자 모드 활성화.
4. "압축 해제된 확장 프로그램을 로드합니다"를 클릭하고 적절한 브라우저 폴더(`browsers/chrome`, `browsers/edge` 또는 `browsers/firefox`)를 선택.

### 2. API 키 받기

최소 하나의 지원되는 클라우드 AI 제공업체의 API 키가 필요합니다. 제공업체의 개발자 콘솔 또는 API 플랫폼에서 키를 생성하세요. 각 제공업체의 키는 확장 프로그램 내에서 별도로 저장됩니다 - 제공업체 간 자유롭게 전환할 수 있습니다.

### 3. 설정 및 빌드

1. 이용약관에 동의 (첫 실행 시 자동으로 표시됩니다).
2. 확장 프로그램 팝업에서 설정 아이콘을 클릭.
3. API 키를 붙여넣고 "키 저장"을 클릭.
4. 드롭다운에서 AI 모델 선택.
5. 채팅에서 빌드하려는 내용 설명.
6. "확장 프로그램 빌드" 또는 "웹사이트 빌드"를 클릭하고 생성 대기.
7. 내장 편집 도구를 사용하여 필요에 따라 생성된 파일 검토 및 편집.
8. "모두 ZIP으로 다운로드" 클릭.
9. 확장 프로그램의 경우: ZIP 압축 해제, `chrome://extensions`로 이동, 개발자 모드 활성화, "압축 해제된 확장 프로그램을 로드합니다" 클릭. 웹사이트의 경우: 압축 해제 후 브라우저에서 `index.html` 열기.

> **기타 브라우저:** 생성된 확장 프로그램은 Manifest V3이며 Edge, Brave, Whale 및 기타 Chromium 기반 브라우저와 호환됩니다. 사이드로딩 단계는 브라우저마다 다릅니다.

## 온디바이스 모델 설정

온디바이스 모델은 API 키나 클라우드 연결 없이 하드웨어에서 완전히 실행됩니다. **이 모델들은 특정 브라우저에서만 사용할 수 있습니다:** Google Chrome에서는 Gemini Nano, Microsoft Edge에서는 Phi-4 Mini. 기타 Chromium 기반 브라우저(Brave, Whale 등)와 Firefox는 현재 브라우저 API를 통한 온디바이스 AI를 지원하지 않습니다.

**클라우드 모델과의 주요 차이점:**
- 온디바이스 모델은 **채팅과 파일 편집**에만 사용할 수 있으며, 전체 빌드에는 사용할 수 없습니다.
- 처음 사용 시 모델 다운로드가 필요합니다 (몇 분이 소요될 수 있습니다).
- 하드웨어 요구 사항이 엄격합니다 - 문제 해결 전에 확인하세요.

### Chrome - Gemini Nano

1. Chrome 버전 127 이상 사용 (Dev 또는 Canary 권장).
2. `chrome://flags/#optimization-guide-on-device-model`로 이동하여 **Enabled BypassPerfRequirement**로 설정.
3. `chrome://flags/#prompt-api-for-gemini-nano`로 이동하여 **Enabled**로 설정.
4. Chrome 재시작.
5. `chrome://on-device-internals`로 이동하여 모델 상태 확인. 모델이 다운로드되지 않은 경우 `chrome://components/`로 이동하여 **Optimization Guide On Device Model**을 찾아 **Check for update** 클릭.
6. 모델 다운로드 완료까지 대기. 몇 분이 소요될 수 있습니다. 다운로드 중에는 Chrome을 열어 두세요.

### Edge - Phi-4 Mini

1. Edge Dev 또는 Canary (버전 138 이상) 사용. Edge 139 이상에는 Phi-4 Mini가 기본 포함되어 있습니다.
2. `edge://flags/`로 이동하여 **Prompt API for Phi mini**를 검색하고 **Enabled**로 설정.
3. 선택적으로 **Enable on device AI model debug logs**를 활성화하여 문제 해결에 활용.
4. Edge 재시작.
5. `edge://on-device-internals`로 이동하여 **Device performance class**가 **High** 이상인지 확인.
6. 모델은 처음 사용할 때 자동으로 다운로드됩니다. 몇 분이 소요될 수 있습니다. 다운로드 중에는 Edge를 열어 두세요.

### 하드웨어 요구 사항

**Edge:** Windows 10/11 또는 macOS 13.3 이상, 20 GB 이상의 여유 저장 공간, 5.5 GB 이상의 VRAM, 종량제가 아닌 인터넷 연결.

**Chrome:** 22 GB 여유 저장 공간, 4 GB 초과 VRAM (GPU) 또는 16 GB 이상 RAM 및 4코어 이상 CPU (CPU 모드), 종량제가 아닌 연결.

> **참고:** 온디바이스 모델은 채팅과 파일 편집에만 사용할 수 있습니다. 확장 프로그램이나 웹사이트의 전체 빌드에는 클라우드 모델을 선택하세요.

## 빌드 모드 이해

NuModeX Ext Maker에는 두 가지 빌드 모드가 있습니다:

**확장 프로그램 빌드** - manifest.json, 팝업 파일, 콘텐츠 스크립트 및 기타 필요한 파일을 포함하는 완전한 Manifest V3 브라우저 확장 프로그램을 생성합니다.

**웹사이트 빌드** - HTML, CSS 및 JavaScript 파일을 포함하는 완전한 정적 웹사이트를 생성합니다.

두 모드 모두 동일한 채팅 인터페이스를 사용합니다. AI는 생성 시 전체 대화 기록을 읽으므로 빌드 전에 여러 메시지를 통해 요구 사항을 구체화할 수 있습니다.

## 생성된 파일 검토

빌드 완료 후 확장 프로그램의 오른쪽 패널에 생성된 파일이 표시됩니다.

**파일 트리** - 생성된 모든 파일이 오른쪽 패널 상단의 클릭 가능한 목록에 표시됩니다. 파일을 클릭하면 내용을 볼 수 있습니다. 파일은 하위 디렉토리 경로(예: `assets/style.css`)를 포함한 이름으로 정리되어 있습니다.

**코드 뷰어** - 파일을 선택하면 파일 트리 아래의 코드 뷰어에 구문 강조와 함께 내용이 표시됩니다. 뷰어는 파일 유형(JavaScript, JSON, HTML, CSS, Markdown)을 자동 감지하고 적절한 강조를 적용합니다. 복사 버튼을 사용하여 파일 내용을 클립보드에 복사할 수 있습니다.

**수동 편집** - 코드 뷰어 상단의 편집 토글을 클릭하여 수동 편집 모드로 전환합니다. 코드 뷰어가 텍스트 편집기로 변경되어 직접 수정할 수 있습니다. 토글을 다시 클릭하면 수동 편집 모드를 종료합니다.

**라이브 미리보기** - 확장 프로그램이나 웹사이트의 시각적 미리보기를 보려면 더 보기 (▾) > 미리보기를 클릭합니다. 샌드박스화된 미리보기가 모달로 열리며 popup.html(확장 프로그램의 경우) 또는 index.html(웹사이트의 경우)을 렌더링합니다. 미리보기는 CSS와 JavaScript 파일을 자동으로 인라인화하여 올바르게 렌더링합니다. 이것은 시각적 전용 미리보기입니다 - 브라우저 확장 프로그램 API(chrome.tabs, chrome.storage 등)와 외부 리소스는 미리보기에서 작동하지 않습니다. 프로젝트에 popup.html이나 index.html이 없으면 미리보기에 정보 메시지가 표시됩니다.

**변경 사항 보기** - AI 편집 후 변경 사항 보기를 클릭하면 수정 전후의 차이를 확인할 수 있습니다. 통합 뷰와 나란히 보기 간에 전환할 수 있습니다. 여러 파일이 변경된 경우 상단의 탭으로 전환할 수 있습니다.

## 빌드 후 편집

첫 번째 빌드 후 여러 편집 옵션을 사용할 수 있습니다:

**파일 편집** - 단일 파일을 선택하고 변경 사항을 설명합니다. 집중적인 수정에 적합합니다.

**파일 추가** - 새 파일을 만들고 포함할 내용을 설명합니다.

**확장 프로그램 개선** - 전체 프로젝트에 걸친 변경 사항을 설명합니다. AI가 여러 파일을 한 번에 수정할 수 있습니다.

**수동 편집** - 코드 뷰어에 직접 클릭하여 코드를 수동으로 편집합니다.

**실행 취소** - 마지막 AI 편집을 되돌립니다. 실행 취소는 1회분만 가능합니다.

## 적절한 모델 선택

특정 모델을 추천하는 대신(자주 변경되므로), 중요한 요소에 따른 선택 방법을 설명합니다:

**컨텍스트 윈도우 크기** - 모델이 한 번에 처리할 수 있는 대화 기록과 코드의 양을 결정합니다. 소수의 파일로 구성된 간단한 확장 프로그램에는 작은 컨텍스트 윈도우로 충분합니다. 복잡한 다중 파일 프로젝트에는 모든 파일과 대화를 한 번에 볼 수 있도록 큰 컨텍스트 윈도우를 가진 모델을 선택하세요.

**출력 토큰 제한** - 모델이 단일 응답에서 생성할 수 있는 코드의 양을 결정합니다. JSON 파싱 오류나 출력 잘림이 발생하면 더 높은 출력 제한을 가진 모델로 전환하세요. 대규모 프로젝트에는 더 많은 토큰을 출력할 수 있는 모델이 필요합니다.

**추론 능력** - 일부 모델은 복잡한 논리, 다단계 지시 및 파일 간 일관성 유지에 더 뛰어납니다. 간단한 팝업 확장 프로그램에는 대부분의 모델이 적합합니다. 콘텐츠 스크립트, 백그라운드 워커 및 복잡한 상호작용이 있는 확장 프로그램에는 강력한 추론 능력을 가진 모델을 선택하세요.

**비용** - 크고 고성능인 모델은 API 호출당 비용이 더 높습니다. 채팅과 간단한 편집에는 작은 모델부터 시작하고, 복잡한 프로젝트의 빌드나 개선이 필요할 때 큰 모델로 전환하세요. 비용은 선택한 모델과 AI 제공업체에 따라 다릅니다. SoraVantia GK는 어떤 AI 제공업체와도 제휴하지 않으며 API 요금의 어떤 부분도 관리하거나 받지 않습니다.

**속도** - 작은 모델은 응답이 더 빠릅니다. 작은 변경 사항을 빠르게 반복하는 경우 빠른 모델이 시간을 절약합니다. 품질이 속도보다 중요한 전체 빌드에는 큰 모델을 사용할 가치가 있습니다.

| 사용 사례 | 확인할 사항 |
|----------|-----------------|
| 간단한 확장 프로그램 (팝업 전용, 작은 스크립트) | 적당한 컨텍스트와 출력 제한을 가진 모든 모델 |
| 복잡한 확장 프로그램 (콘텐츠 스크립트, 백그라운드 워커, 다중 파일) | 큰 컨텍스트 윈도우, 높은 출력 제한, 강력한 추론 능력 |
| 빠른 채팅 질문 및 브레인스토밍 | 모든 모델 - 출력 크기보다 속도가 중요 |
| 단일 파일 편집 | 온디바이스 모델을 포함한 모든 모델 (무료, API 키 불필요) |
| 비용 중심 개발 | 채팅에는 소형 또는 중형 모델, 빌드에만 대형 모델 |

AI 제공업체의 모델 문서에서 현재 컨텍스트 윈도우 크기, 출력 제한 및 가격을 확인하세요.

## 최상의 결과를 위한 팁

- 간단한 설명부터 시작하여 점진적으로 구축. 먼저 핵심 기능을 설명한 다음 편집 및 개선을 사용하여 점진적으로 기능 추가.
- 복잡한 프로젝트에는 컨텍스트 윈도우가 큰 모델 사용. 큰 모델은 작은 모델보다 큰 출력을 더 잘 처리합니다.
- "확장 프로그램 파일을 추출할 수 없습니다"가 표시되면 프롬프트가 한 번의 생성에 너무 복잡합니다. 초기 프롬프트를 단순화하고 편집을 통해 기능 추가.
- JSON 파싱 오류가 표시되면 모델의 응답이 너무 길어 잘렸습니다. 더 간단한 프롬프트를 시도하거나 출력 제한이 큰 모델로 전환.
- 클라우드, 사용자 정의, 원격 모델 모두 빌드, 편집, 채팅에 사용 가능. 필요와 예산에 가장 적합한 모델 선택.
- 온디바이스 모델은 채팅 및 편집에 작동하지만 전체 확장 프로그램이나 웹사이트를 빌드할 수 없습니다. 빌드에는 클라우드 또는 사용자 정의 모델 사용.
- Enter로 채팅 메시지 전송. Shift+Enter로 줄바꿈. Ctrl/Cmd+Enter로 확장 프로그램 빌드. Ctrl/Cmd+Shift+Enter로 웹사이트 빌드.
- 빌드 후 단일 파일 변경에는 파일 편집, 다중 파일 변경에는 확장 프로그램 개선 사용.
- 더 보기 (▾) > 파일 가져오기를 통해 기존 파일을 가져와서 AI로 편집.

## 문제 해결

**문제 해결 전에 다음 리소스를 확인하세요:**
- 클라우드 AI 제공업체의 API 문서에서 현재 모델 제한, 가격 및 상태 확인.
- 브라우저의 개발자 문서에서 최신 온디바이스 AI 요구 사항 및 알려진 문제 확인.
- 위의 [온디바이스 모델 설정](#온디바이스-모델-설정) 섹션에서 플래그 및 하드웨어 요구 사항 확인.

### "API 키가 설정되지 않았습니다"
설정을 열고 선택한 제공업체의 API 키를 붙여넣으세요. 각 제공업체에는 자체 키가 있습니다 - 선택한 모델의 제공업체 키를 저장했는지 확인하세요.

### "확장 프로그램 파일을 추출할 수 없습니다"
AI 응답이 너무 복잡하거나 형식이 잘못되었습니다. 다음을 시도하세요:
- 프롬프트를 단순화 (한 번에 적은 기능 설명)
- 더 큰 출력 제한을 가진 모델로 전환
- 기본 버전을 먼저 빌드한 다음 "확장 프로그램 개선"으로 기능 추가

### "JSON 파싱 오류"
모델의 응답이 너무 길어 JSON이 완성되기 전에 잘렸습니다. 다음을 시도하세요:
- 더 간단한 프롬프트
- 더 높은 출력 토큰 제한을 가진 모델

### 온디바이스 모델이 "다운로드 중"에서 멈춤
일반적인 문제입니다. 다음을 확인하세요:
1. **하드웨어 요구 사항을 충족하나요?** `edge://on-device-internals` (Edge) 또는 `chrome://on-device-internals` (Chrome)로 이동하여 디바이스 클래스를 확인.
2. **플래그가 활성화되어 있나요?** 위의 [온디바이스 모델 설정](#온디바이스-모델-설정) 섹션을 참조.
3. **충분한 저장 공간이 있나요?** Edge는 20 GB, Chrome은 22 GB의 여유 공간이 필요합니다.
4. **종량제 연결인가요?** 종량제/데이터 제한 연결에서는 모델이 다운로드되지 않습니다.
5. **브라우저를 열어 두었나요?** 브라우저를 닫으면 다운로드가 중단됩니다.
6. **브라우저를 재시작하고** 5-10분 기다려 보세요.

### 로드 후 확장 프로그램이 작동하지 않음
- 브라우저 콘솔 (`chrome://extensions` > 확장 프로그램의 "오류" 클릭)에서 오류 메시지 확인.
- 올바른 폴더 (manifest.json이 포함된 폴더)를 로드했는지 확인.
- 정확한 동작을 지정하는 더 자세한 프롬프트로 재생성 시도.

### 사용자 정의 모델이 응답하지 않음
- 엔드포인트 URL이 올바르고 접근 가능한지 확인.
- 서버가 `/v1/chat/completions` API 형식을 지원하는지 확인.
- 서버에 API 키가 필요한지, 제공했는지 확인.
- 프롬프트가 표시되면 확장 프로그램 호스트 권한을 허용.
