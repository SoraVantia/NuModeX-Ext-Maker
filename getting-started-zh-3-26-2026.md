# NuModeX Ext Maker 入门指南

本指南将比 README 更详细地引导您完成设置、首次构建和常见问题。

## 前提条件

- 基于 Chromium 的浏览器（Chrome、Edge、Brave、Whale）或 Firefox
- 至少一个云端 AI 提供商的 API 密钥 - 如果使用设备端模型则不需要
- 浏览器扩展程序的基本知识（有帮助但非必需）

## 云端模型设置

### 1. 安装扩展程序

**从浏览器扩展程序商店（即将推出）：**
NuModeX Ext Maker 尚未在任何浏览器扩展程序商店上架。商店上架信息将在 [numodex.com/numodexextmaker](https://numodex.com/numodexextmaker) 和 GitHub Releases 上公布。

**从源代码（开发者模式）：**
1. 克隆或下载存储库。
2. 打开 `chrome://extensions`（或您浏览器中的对应页面）。
3. 启用开发者模式。
4. 点击"加载已解压的扩展程序"并选择相应的浏览器文件夹（`browsers/chrome`、`browsers/edge` 或 `browsers/firefox`）。

### 2. 获取 API 密钥

您需要至少一个受支持的云端 AI 提供商的 API 密钥。访问您提供商的开发者控制台或 API 平台来生成密钥。每个提供商的密钥在扩展程序中单独保存 - 您可以自由切换提供商。

### 3. 配置和构建

1. 接受服务条款（首次启动时自动显示）。
2. 点击扩展程序弹出窗口中的设置图标。
3. 粘贴您的 API 密钥并点击"保存密钥"。
4. 从下拉菜单中选择 AI 模型。
5. 在聊天中描述您想要构建的内容。
6. 点击"构建扩展程序"或"构建网站"并等待生成。
7. 使用内置编辑工具根据需要审查和编辑生成的文件。
8. 点击"全部下载为 ZIP"。
9. 对于扩展程序：解压 ZIP，前往 `chrome://extensions`，启用开发者模式，点击"加载已解压的扩展程序"。对于网站：解压并在浏览器中打开 `index.html`。

> **其他浏览器：** 生成的扩展程序是 Manifest V3，与 Edge、Brave、Whale 及其他基于 Chromium 的浏览器兼容。侧载步骤因浏览器而异。

## 设备端模型设置

设备端模型完全在您的硬件上运行，无需 API 密钥或云连接。**这些模型仅在特定浏览器中可用：** Google Chrome 中的 Gemini Nano 和 Microsoft Edge 中的 Phi-4 Mini。其他基于 Chromium 的浏览器（Brave、Whale 等）和 Firefox 目前不支持通过浏览器 API 使用设备端 AI。

**与云端模型的主要区别：**
- 设备端模型仅可用于**聊天和文件编辑**，不能用于完整构建。
- 首次使用时需要下载模型（可能需要几分钟）。
- 硬件要求严格 - 在排除故障前请先检查。

### Chrome - Gemini Nano

1. 使用 Chrome 版本 127 或更高（推荐使用 Dev 或 Canary 以获得最佳效果）。
2. 前往 `chrome://flags/#optimization-guide-on-device-model` 并设置为 **Enabled BypassPerfRequirement**。
3. 前往 `chrome://flags/#prompt-api-for-gemini-nano` 并设置为 **Enabled**。
4. 重启 Chrome。
5. 前往 `chrome://on-device-internals` 并验证模型状态。如果模型未下载，前往 `chrome://components/`，找到 **Optimization Guide On Device Model** 并点击 **Check for update**。
6. 等待模型下载完成。这可能需要几分钟。下载期间请保持 Chrome 打开。

### Edge - Phi-4 Mini

1. 使用 Edge Dev 或 Canary（版本 138+）。Edge 139+ 默认包含 Phi-4 Mini。
2. 前往 `edge://flags/` 并搜索 **Prompt API for Phi mini**，设置为 **Enabled**。
3. 可选：启用 **Enable on device AI model debug logs** 以进行故障排除。
4. 重启 Edge。
5. 前往 `edge://on-device-internals` 并验证您的 **Device performance class** 为 **High** 或更高。
6. 模型会在首次使用时自动下载。这可能需要几分钟。下载期间请保持 Edge 打开。

### 硬件要求

**Edge：** Windows 10/11 或 macOS 13.3+，至少 20 GB 可用存储空间，5.5 GB+ VRAM，以及非流量计费的互联网连接。

**Chrome：** 22 GB 可用存储空间，超过 4 GB VRAM（GPU）或 16 GB+ RAM 及 4+ CPU 核心（CPU 模式），以及非流量计费的连接。

> **注意：** 设备端模型仅可用于聊天和文件编辑。要构建完整的扩展程序或网站，请选择云端模型。

## 了解构建模式

NuModeX Ext Maker 有两种构建模式：

**构建扩展程序** - 生成包含 manifest.json、弹出窗口文件、内容脚本和其他必需文件的完整 Manifest V3 浏览器扩展程序。

**构建网站** - 生成包含 HTML、CSS 和 JavaScript 文件的完整静态网站。

两种模式使用相同的聊天界面。AI 在生成时会读取您的整个对话历史，因此您可以在构建前通过多条消息来完善您的需求。

## 查看生成的文件

构建完成后，扩展程序的右侧面板会显示生成的文件。

**文件树** - 所有生成的文件显示在右侧面板顶部的可点击列表中。点击任何文件即可查看其内容。文件按名称组织，包括子目录路径（例如 `assets/style.css`）。

**代码查看器** - 选择文件后，其内容会在文件树下方的代码查看器中以语法高亮显示。查看器会自动检测文件类型（JavaScript、JSON、HTML、CSS、Markdown）并应用适当的高亮。您可以使用复制按钮将任何文件的内容复制到剪贴板。

**手动编辑** - 点击代码查看器上方的编辑切换按钮进入手动编辑模式。代码查看器会变为文本编辑器，您可以直接手动进行更改。再次点击切换按钮退出手动编辑模式。

**实时预览** - 要查看扩展程序或网站的可视化预览，请点击更多 (▾) > 预览。这将打开一个带有沙盒化预览的模态窗口，渲染您的 popup.html（扩展程序）或 index.html（网站）。预览会自动内联您的 CSS 和 JavaScript 文件以正确渲染。请注意，这只是视觉预览 - 浏览器扩展程序 API（如 chrome.tabs、chrome.storage）和外部资源在预览中不可用。如果您的项目没有 popup.html 或 index.html，预览将显示一条信息消息。

**查看更改** - AI 编辑后，点击查看更改可以看到修改前后的对比。您可以在统一视图和并排视图之间切换。如果修改了多个文件，顶部的标签页可以让您在它们之间切换。

## 构建后编辑

首次构建后，您有多种编辑选项：

**编辑文件** - 选择单个文件并描述更改。适合有针对性的修复。

**添加文件** - 创建新文件并描述其应包含的内容。

**改进扩展程序** - 描述跨整个项目的更改。AI 可以一次修改多个文件。

**手动编辑** - 直接点击代码查看器手动编辑代码。

**撤销** - 撤销上一次 AI 编辑。仅支持一级撤销。

## 选择合适的模型

我们不推荐特定模型（因为它们经常变化），而是说明如何根据重要因素进行选择：

**上下文窗口大小** - 决定模型一次可以处理多少对话历史和代码。对于只有少量文件的简单扩展程序，较小的上下文窗口就足够了。对于复杂的多文件项目，请选择具有更大上下文窗口的模型，以便一次查看所有文件和对话。

**输出令牌限制** - 决定模型在单次响应中可以生成多少代码。如果出现 JSON 解析错误或输出被截断，请切换到具有更高输出限制的模型。较大的项目需要能够输出更多令牌的模型。

**推理能力** - 一些模型在复杂逻辑、多步骤指令和跨文件保持一致性方面更出色。对于简单的弹出窗口扩展程序，大多数模型都能胜任。对于具有内容脚本、后台 Worker 和复杂交互的扩展程序，请选择以强大推理能力著称的模型。

**成本** - 更大、更强大的模型每次 API 调用的成本更高。聊天和简单编辑从较小的模型开始，在需要构建或改进复杂项目时再切换到更大的模型。成本因您选择的模型和 AI 提供商而异。SoraVantia GK 不与任何 AI 提供商有关联，也不控制或收取 API 费用的任何部分。

**速度** - 较小的模型响应更快。如果您正在快速迭代小更改，快速模型可以节省时间。对于质量比速度更重要的完整构建，更大的模型值得等待。

| 使用场景 | 应关注的要素 |
|----------|-----------------|
| 简单扩展程序（仅弹出窗口、小脚本） | 具有适度上下文和输出限制的任何模型 |
| 复杂扩展程序（内容脚本、后台 Worker、多文件） | 大上下文窗口、高输出限制、强推理能力 |
| 快速聊天问题和头脑风暴 | 任何模型 - 速度比输出大小更重要 |
| 单文件编辑 | 包括设备端在内的任何模型（免费，无需 API 密钥） |
| 注重预算的开发 | 聊天使用小型或中型模型，仅构建时使用大型模型 |

请查阅您的 AI 提供商的模型文档，了解当前的上下文窗口大小、输出限制和定价。

## 获得最佳效果的提示

- 从简单描述开始逐步构建。先描述核心功能，然后使用编辑和改进逐步添加更多功能。
- 复杂项目使用上下文窗口更大的模型。较大的模型比较小的模型更好地处理较大的输出。
- 如果看到"无法提取扩展程序文件"，说明提示对于一次生成来说太复杂了。简化初始提示并通过编辑添加功能。
- 如果看到 JSON 解析错误，说明模型的响应太长被截断了。尝试更简单的提示或切换到输出限制更大的模型。
- 云端、自定义和远程模型都可以用于构建、编辑和聊天。选择最适合您需求和预算的模型。
- 设备端模型可用于聊天和编辑，但无法构建完整的扩展程序或网站。构建请使用云端或自定义模型。
- Enter 发送聊天消息。Shift+Enter 换行。Ctrl/Cmd+Enter 构建扩展程序。Ctrl/Cmd+Shift+Enter 构建网站。
- 构建后，使用编辑文件进行单文件更改，使用改进扩展程序进行跨多个文件的更改。
- 通过更多 (▾) > 导入文件导入现有文件以使用 AI 编辑。

## 故障排除

**在排除故障前，请查阅以下资源：**
- 您的云端 AI 提供商的 API 文档，了解当前模型限制、定价和状态。
- 您浏览器的开发者文档，了解最新的设备端 AI 要求和已知问题。
- 上方的[设备端模型设置](#设备端模型设置)部分，了解标志和硬件要求。

### "API 密钥未设置"
打开设置并粘贴您所选提供商的 API 密钥。每个提供商都有自己的密钥 - 确保您保存了所选模型对应提供商的密钥。

### "无法提取扩展程序文件"
AI 的响应过于复杂或格式不正确。请尝试：
- 简化您的提示（一次描述较少的功能）
- 切换到具有更大输出限制的模型
- 先构建一个基本版本，然后使用"改进扩展程序"添加功能

### "JSON 解析错误"
模型的响应太长，在 JSON 完成之前被截断。请尝试：
- 更简单的提示
- 具有更高输出令牌限制的模型

### 设备端模型卡在"下载中"
这是一个常见问题。请检查：
1. **满足硬件要求吗？** 前往 `edge://on-device-internals`（Edge）或 `chrome://on-device-internals`（Chrome）并验证您的设备类别。
2. **标志已启用吗？** 请参阅上方的[设备端模型设置](#设备端模型设置)部分。
3. **存储空间足够吗？** Edge 需要 20 GB 可用空间，Chrome 需要 22 GB 可用空间。
4. **流量计费连接？** 模型不会在流量计费/数据限制的连接上下载。
5. **浏览器保持打开了吗？** 关闭浏览器会停止下载。
6. **尝试重启浏览器**并等待 5-10 分钟。

### 加载后扩展程序不工作
- 检查浏览器控制台（`chrome://extensions` > 点击扩展程序上的"错误"）查看错误消息。
- 确保加载了正确的文件夹（包含 manifest.json 的文件夹）。
- 尝试使用更详细的提示重新生成，指定确切的行为。

### 自定义模型不响应
- 验证端点 URL 是否正确且可访问。
- 确认服务器支持 `/v1/chat/completions` API 格式。
- 检查服务器是否需要 API 密钥以及您是否提供了。
- 在提示时授予扩展程序主机权限。
